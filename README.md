# 南山置业图鉴 · Nanshan Property Atlas

> 把深圳南山豪宅的"区位价值"从纯销售话术，变成**可溯源的距离测算 + 会权衡的 AI 选房顾问**——事实层用真实坐标算、可验证；判断层明确标注为主观。
> 真实坐标 × 真 3D × 真 AI 的购房辅助工具。进门选「买家 / 经纪」双模式。

**🔗 在线 Demo：** https://nieears-cell.github.io/nanshan-atlas/

> ⚠️ 定位说明：这是一个**面向大众购房者的辅助决策 demo / 求职作品集**，不是面向超高净值客户的成交工具。它**不涉及** off-market 房源、议价情报、产权尽调、持有/税务结构——那是持牌机构的业务，不在本作品范围内。评分仅供参考，**不构成投资建议**。

## Problem · 为什么做
普通购房者在售楼处只能看平面户型、听销售话术，**难以横向比较多个楼盘**的区位 / 教育 / 医疗 / 通勤。"一线海景""顶级学区"都是话术。本工具尝试把**能用真实坐标算清楚的那部分**(距离)做成可溯源的数字，把**需要判断的那部分**交给会讲清取舍的 AI，并诚实区分两者。

## Constraints · 真实世界的硬墙
- 中国大陆禁用谷歌实景三维；照片级贴图楼需付费倾斜摄影 → 免费数据天花板是 OSM 体块。
- 高德 / 天地图是 GCJ-02 坐标系，与 GPS/WGS-84 差约 500m，直接叠会错位。
- 无后端、单文件、纯前端 → 所有能力在浏览器里跑。
- 大模型会幻觉：让它直接输出楼盘，会编出不存在的标的。

## Solution · 三大能力
- 🤖 **AI 选房顾问**（Google Gemini 2.5 Flash）：自然语言需求 → 多约束**非线性权衡**（命中 / 让步 + 整体取舍）。用 `responseSchema` + 闭集 enum **把推荐的楼盘 id 锁死在清单内**，结构性杜绝"编造不存在的楼盘"这一类幻觉。
- 🏔 **3D 山海格局**（CesiumJS）：真实地形 + 天地图 / Esri 卫星 + OSM 立体楼块 + 入场导览 + 可分享视角。含 GCJ-02→WGS-84 坐标纠偏。楼为无贴图体块（数据天花板），主要用于直观感受"背山面海"的宏观格局，**不用于楼层/朝向/遮挡这类微观决策**。
- 📊 **距离测算（事实层）+ 五维评分（含判断层）**：
  - **事实层(可验证)**：到最近学校 / 医院 / 地铁的**直线距离**，用真实坐标程序化计算、可溯源。
  - **判断层(主观)**：保值率 / 健康宜居等为**编辑主观估值**，已在界面标注，不应与实测距离等同看待。

## Tradeoffs · 关键产品取舍
- **为什么不让 AI 直接输出楼盘？** 会编造不存在的标的 → 用 schema 锁死 id 枚举（id 级杜绝幻觉），距离从真实坐标算、不问 LLM；AI 只负责权衡与解释。
- **为什么非线性权衡？** 线性加权是伪精确；价值在说清"哪些能全得、哪些必须二选一"。
- **为什么楼是色块？** 照片级实景在中国要付费授权，作品集阶段不值；诚实标注"基于公开建筑数据估算"。
- **人工兜底**：用户可自己看原始距离数字，不必信 AI。

## Evaluation · 给 AI 功能做了评测（不只是 demo）
内置 AI 选房 Eval（控制台 `nspEval()` 或 AI 面板「🧪 跑 AI 评测」）：**20 例（16 真实诉求 + 4 对抗陷阱）**，对 Gemini 推荐**程序化判分**。

| 指标 | 结果 | 诚实口径 |
|---|---|---|
| **闭集 id 合规率** | **100%**（66 条推荐 0 编造楼盘） | 这是 `responseSchema` enum 的**结构性必然**，证明"不会编造不存在的楼盘"，**不等于** AI 的事实陈述都对 |
| **对抗陷阱通过** | **4 / 4** | 不存在盘不编造 / 不可能预算如实说明 / 矛盾诉求点破取舍 / 跑题不硬推 |
| 优先级命中 | 10 / 11 | 对照透明规则基线 |
| 平均精确率 / 召回率 | 0.69 / 0.64 | 对照"可负担∩按优先级 Top4"规则基线，非绝对真值 |

> **重要：不夸大。** 这里的"0 编造"是 enum 约束的必然结果，证明的是 id 级安全；它**不能**保证 AI 引用的距离/楼高等事实都准确（这些底层数据本身含估算）。要做"事实层准确率"，ground truth 必须来自独立可信源（官方网签 / 第三方测绘），这是本 demo 还没做的。把它如实讲清楚，本身是产品诚实度的一部分。

## 自我评审与边界（这才是产品成熟度）
我做完后，用 **6 个世界级豪宅机构视角**（Savills 级顾问 / HNWI 买家 / RICS 估值师 / 奢侈品牌总监 / 本地资深经纪 / 法务合规）对它做了一轮**对抗式评审**，并据此修正。它教会我的几条边界：
- **它量化的是大众购房在乎的距离，不是顶豪在乎的稀缺/圈层/栋位/海景视廊**——所以定位为大众辅助工具，不冒充顶豪产品。
- **"距三甲""学区分"用的是直线距离**：深圳实际入学受**学位锁定 + 积分 + 大学区**约束，与直线距离关系有限；距三甲也≠医疗质量。这些已在"诚实的局限"标注。
- **判断层分数(保值率等)是主观估值**，不是估值报告；任何前瞻性表述都不构成投资建议。
- 评审里"补 off-market / 议价 / 尽调 / 持有结构"等是**持牌机构的业务，刻意不做**——知道边界在哪，比假装无所不能更专业。

## 技术栈 / 怎么做的
高德 JS API 2.0（2D）· CesiumJS 1.142（3D）· 天地图 / Esri · Turf.js · Google Gemini REST。单文件 HTML、零构建、CDN、全程优雅降级。
**开发方式：** AI 结对编程 —— 产品 / 架构 / 数据 / 防幻觉设计与全程调试由我主导，AI 负责实现代码。

## 使用
AI 功能需访问者**自带 Gemini API key**（仅存浏览器 localStorage，不上传）。点右下「✦ AI 选房顾问」→ ⚙ 设置 key。

## Limitations · 诚实的局限
- **数据**：单价/总价/面积为公开渠道人工整理（截至 2026.04，非实时）；楼高为 OSM 公开估算；部分坐标手工校正；在建期房用备案价，不代表成交价。
- **方法**：教育/医疗为**直线距离**(非真实路网/非学位可入读判断)；五维中保值率/健康宜居/空气为**主观或代理指标**，非独立估值。
- **范围**：仅南山 19 个标杆盘；卫星影像 z18 封顶；3D 楼为色块，不可用于楼层/朝向/遮挡决策。
- **AI**：建议仅供参考、需实勘核实；底层数据若有误，AI 会照样流畅复述——务必交叉核验。
- **不构成投资建议**，不替代专业法律/税务/估值意见。

真实路网等时圈替代直线距离 · 事实层准确率评测(接独立数据源) · 判断分锚定可证伪指标(成交CAGR/租售比/流动性) · 楼盘对比。
